# Правила разработки
1. **Ветки**: Солздавайте новуюветку для каждой задачи (например, `feature/task-name`).
2. **Коммиты**: Пишите краткие и понятные сообщения к коммитам
3. **pull Requests**: Все изменения вносятся через PR. Перед слиянием нужно получить одобрение (Approve) от ответственного разработчика.
